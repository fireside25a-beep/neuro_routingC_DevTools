NeuroRoute DevCLI is a portable C11 developer command-line tool built around a deterministic routing library. The project preserves the original NeuroRoute behavior, including maximum-weight graph routing, activation thresholds, fallback handling, atomic route-use counters, and 256-bit Hamming-distance spatial matching, while adding a practical developer workflow for common programming languages.

The DevCLI can inspect a directory, detect the project type using deterministic priority rules, report ambiguous layouts instead of guessing, identify installed compilers and interpreters, create minimal starter projects, and safely run build, run, test, clean, and edit operations. Supported languages include C, C++, Python, Rust, JavaScript, Go, Zig, and Java. Missing toolchains are reported clearly and are never treated as successful tests.

Process execution uses POSIX argument arrays with posix_spawnp and waitpid. The project does not pass user input through system, popen, sh -c, or bash -c, reducing command-injection risk in the process-launching layer. NeuroRoute DevCLI is not a shell, terminal emulator, compiler, package manager, operating system, or AI model. It is a focused orchestration tool for local developer workflows.

The repository includes the portable C11 library, the new CLI, unit tests, CLI smoke tests, starter-project generators, build rules, sanitizer targets, documentation, and a self-contained installer. The installer embeds the full source release in one shell script, verifies its payload, builds the project, runs tests, checks the original demo, performs a generated C-project smoke test, and installs the final neuro executable.

The release was validated on Debian GNU/Linux x86_64 using both GCC and Clang with strict C11 warnings enabled. Runtime starter-project tests passed for C, C++, Python, JavaScript, Go, and Java. Rust and Zig were marked skipped because those toolchains were unavailable in the validation environment. The source is designed for macOS and Linux, with no architecture-specific binaries or handwritten assembly included.

Project generation is intentionally conservative: existing non-empty directories are protected from accidental overwrites, generated programs print a known verification message, and each external command returns its real exit status. The repository is suitable for developers who want a small, inspectable, source-first tool that combines deterministic project detection with straightforward access to locally installed development toolchains without hidden behavior.
